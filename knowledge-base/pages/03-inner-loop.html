Inner Loop

Model Weight Optimization

Optimizes how the model performs — updating weights via SFT, DPO, and GRPO from production traces. Runs periodically, producing improved checkpoints deployed via KServe canary.

2 Training Approaches

Supervised Fine-Tuning POC Recommended

When to Use

Initial adaptation to task domain, bootstrapping from production traces. The starting point for any fine-tuning pipeline.

Data Format

Clean (input, output) pairs from high-quality traces of successful agent completions.

Strengths

  • + Simple and well-understood
  • + Works well for tool-calling format compliance
  • + Low data requirements relative to RL methods

Weaknesses

  • − Doesn't teach what NOT to do
  • − Can overfit to surface patterns
  • − Poor at teaching multi-step reasoning
For RFE Creator

SFT is your starting point. Collect production traces, filter for high-quality completions, and use SFT to teach the model the task format and domain before applying DPO or GRPO refinement.

Direct Preference Optimization

When to Use

After SFT, to refine behavior based on quality distinctions. Teaches the model to prefer good outputs over bad ones.

Data Format

Preference pairs: (prompt, chosen_response, rejected_response) with labels. M-DPO (ICLR 2025) extends to multi-turn agent conversations.

Strengths

  • + No reward model needed (simpler than PPO)
  • + Direct bridge from eval-harness pairwise judges
  • + Teaches quality preferences, not just imitation

Weaknesses

  • − Requires good preference pairs (quality-sensitive)
  • − Credit assignment across turns is hard
  • − Can amplify biases in preference data
For RFE Creator

DPO maps directly to eval-harness pairwise judges via compare_runs(). Winner = chosen, loser = rejected. This is the bridge between outer loop evaluation and inner loop training.

Group Relative Policy Optimization DeepSeek-R1

When to Use

After SFT+DPO, for reinforcement from verifiable rewards. Natural fit for agent tasks with measurable outcomes.

Data Format

Prompts + reward function. Generate G responses per prompt → score all → normalize advantages within group → update policy.

Strengths

  • + No critic model needed (saves ~50% memory)
  • + Used to train DeepSeek-R1
  • + Works with verifiable rewards (tool success, cost)

Weaknesses

  • − Zero gradient when group is homogeneous
  • − Needs many samples per prompt (compute-heavy)
  • − NGRPO (2025) needed for advantage calibration
For RFE Creator

GRPO with eval-harness judges as reward functions is the most advanced stage. Bool judges → pass=1.0/fail=0.0 rewards. Numeric judges → continuous reward signal. This is Sprint 3+ territory.

Low-Rank Adaptation Parameter-Efficient

When to Use

When full fine-tuning is too expensive or when you need task-specific adapters that can be swapped at inference. Especially useful for tool-use recovery in compressed SLMs.

Data Format

Same as SFT/DPO — the difference is in how weights are updated (low-rank decomposition of weight matrices, typically rank 8-64).

Strengths

  • + ~1% of parameters updated (much less memory)
  • + Multiple adapters can be swapped at inference
  • + Targeted tool-use LoRA to recover SLM fidelity

Weaknesses

  • − Limited capacity compared to full fine-tuning
  • − Rank selection requires experimentation
  • − Adapter composition can degrade quality
For RFE Creator

LoRA is critical in the progressive compression pipeline. When distilling to a Granite 8B, tool-calling fidelity degrades fastest. A targeted tool-use LoRA can recover JSON schema compliance and parameter typing accuracy.

5 Structured Agent Distillation (SAD)

SAD segments agent trajectories into REASON and ACT spans, applying segment-specific losses. This outperforms naive token-level distillation for agent tasks.

6 Agent-as-Annotators Pattern

Use a frontier model (Gemini/Claude) as a teacher to generate high-quality agent trajectories. Filter aggressively — the TOUCAN research recommends a 22%+ rejection rate.

Key Results
Fine-tuned 9B student 41.5%
Claude 3.5 Sonnet (teacher) 36.0%

WebArena benchmark — student surpassed teacher

The Pattern
  1. 1. Frontier model generates diverse trajectories
  2. 2. Aggressive quality filtering (22%+ rejection)
  3. 3. Multi-teacher diversity (avoid single-model bias)
  4. 4. SFT on filtered trajectories
  5. 5. Eval-gated deployment
Why It Works

Small models can exceed teacher performance on specific tasks because they focus all capacity on the narrow domain, while the teacher spreads capacity across all domains. Task-specific fine-tuning concentrates the model's learning.
