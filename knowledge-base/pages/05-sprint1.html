🔀 Cross-Cutting Concerns

🚨

Loop Interference

WARNING

If both loops (outer prompt optimization + inner weight optimization) run simultaneously, they can fight each other. The outer loop optimizes prompts for the current model weights — but if the inner loop changes those weights, the optimized prompts may become suboptimal or harmful.

Mitigation: Version-lock the model checkpoint when running outer-loop eval passes. Never run both loops against the same model version simultaneously. Gate weight updates on eval regression checks.
ℹ️

Eval Reliability Protocol

INFO

LLM judge scores have inherent variance (temperature, prompt sensitivity, model mood). A single eval run is not reliable enough to make training decisions.

Recommended protocol:

3x runs → median score → report variance

If variance across 3 runs exceeds 15%, investigate the judge prompt or increase sample size.
